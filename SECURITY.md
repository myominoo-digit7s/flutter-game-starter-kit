# Security Policy

## Supported Versions

This project is currently maintained on the `main` branch.

Security updates and fixes will generally be applied to the latest version of the project.

| Version | Supported |
| ------- | --------- |
| main    | Yes       |

## Reporting a Security Vulnerability

If you discover a security vulnerability in **Flutter + Flame Game Starter Kit**, please do not open a public GitHub issue.

Instead, please report it privately by contacting the maintainer:

```text
myomin.oo@digit7s.com
```

Please include as much information as possible, such as:

* A clear description of the issue
* Steps to reproduce the issue
* Affected files or features
* Possible impact
* Suggested fix, if you have one
* Screenshots, logs, or proof of concept, if applicable

## What to Report

Security issues may include, but are not limited to:

* Exposed API keys, secrets, or credentials
* Unsafe handling of user data
* Insecure storage usage
* Misconfigured AdMob or production identifiers
* Dependency vulnerabilities
* Platform-specific security concerns for Android or iOS
* Any issue that could negatively affect users or developers using this starter kit

## What Not to Report as Security Issues

Please use normal GitHub issues for general bugs or feature requests, such as:

* UI bugs
* Gameplay bugs
* Documentation improvements
* Feature suggestions
* Performance improvements
* Code refactoring suggestions

## Response Process

After a security report is received, the maintainer will try to:

1. Review the report
2. Confirm whether the issue is valid
3. Investigate the impact
4. Prepare a fix if needed
5. Publish the fix or mitigation
6. Credit the reporter if appropriate and if they agree

Response time may vary depending on the complexity of the issue and project availability.

## Responsible Disclosure

Please give the maintainer reasonable time to review and fix the issue before publicly sharing details.

Do not use the vulnerability to access, modify, or delete data that does not belong to you.

Do not exploit the issue beyond what is necessary to confirm the vulnerability.

## Dependencies

This project uses Flutter, Flame, and other open-source packages.

If you find a vulnerability in a third-party dependency, please also report it to the maintainers of that dependency when appropriate.

## Secrets and Credentials

This project should not contain real production secrets, API keys, private tokens, or production AdMob unit IDs.

If you accidentally find any sensitive credentials in the repository, please report them privately as soon as possible.

## Thank You

Thank you for helping keep Flutter + Flame Game Starter Kit safe and trustworthy for the community.
