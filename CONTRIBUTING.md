# Contributing to Flutter + Flame Game Starter Kit

Thank you for your interest in contributing to **Flutter + Flame Game Starter Kit**.

This project is an open-source starter kit for building 2D mobile games with Flutter and the Flame game engine. Contributions, suggestions, bug reports, and improvements are welcome.

## How You Can Contribute

You can contribute in many ways:

* Report bugs
* Suggest new features
* Improve documentation
* Add examples or sample games
* Improve code structure
* Fix UI, performance, or platform-specific issues
* Improve tests
* Help with localization

## Before You Start

Please make sure you have the following installed:

* Flutter SDK, latest stable version recommended
* Dart SDK
* Android Studio, VS Code, or your preferred IDE
* Git

## Project Setup

Clone the repository:

```bash
git clone https://github.com/myominoo-digit7s/flutter-game-starter-kit.git
cd flutter-game-starter-kit
```

Install dependencies:

```bash
flutter pub get
```

Run the project:

```bash
flutter run
```

Run tests:

```bash
flutter test
```

Run static analysis:

```bash
flutter analyze
```

## Development Guidelines

Please follow these guidelines when contributing:

* Keep the code clean, readable, and maintainable.
* Follow Flutter and Dart best practices.
* Keep features modular and easy to understand.
* Avoid adding unnecessary dependencies.
* Do not commit generated, temporary, or local environment files.
* Use meaningful names for files, classes, methods, and variables.
* Update documentation when adding or changing features.
* Add or update tests when possible.

## Branch Naming

Please use clear branch names.

Examples:

```text
feature/add-new-game-example
fix/audio-toggle-issue
docs/update-admob-guide
refactor/game-shell-structure
```

## Commit Messages

Use simple and meaningful commit messages.

Examples:

```text
Add basic game example
Fix audio settings toggle
Update localization documentation
Refactor game shell initialization
```

## Pull Request Process

Before opening a pull request, please make sure:

1. Your code runs successfully.
2. `flutter analyze` passes without major issues.
3. `flutter test` passes.
4. Your changes are focused on one topic or issue.
5. Documentation is updated if needed.

When opening a pull request, please include:

* A short summary of the changes
* Why the change is needed
* Screenshots or videos for UI/gameplay changes, if applicable
* Any testing notes

## Reporting Bugs

When reporting a bug, please include:

* A clear description of the issue
* Steps to reproduce
* Expected behavior
* Actual behavior
* Flutter version
* Device or platform information
* Screenshots, logs, or error messages if available

## Suggesting Features

Feature suggestions are welcome.

When suggesting a feature, please explain:

* What problem it solves
* Why it would be useful
* Any possible implementation idea
* Whether it should be part of the starter kit or an example game

## Code Style

This project aims to stay simple, practical, and beginner-friendly.

Please avoid overly complex patterns unless they provide clear value to the starter kit.

## AdMob and Production IDs

This project uses test AdMob IDs by default.

Please do not submit pull requests that include real production AdMob IDs, private API keys, secrets, or credentials.

## Localization

If you add or update text in the app, please make sure localization files are updated when needed.

Current localization support includes:

* English
* Myanmar
* Thai

## Security

Please do not open a public issue for sensitive security problems.

If you discover a security issue, please contact the maintainer privately.
